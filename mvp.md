# Функциональные требования MVP
## Основные модули системы

### Модуль "Главная страница и навигация"
1.	Система должна отображать главную страницу с баннером, содержащим ключевое предложение агентства
2.	Система должна предоставлять навигационное меню с разделами: "Главная", "Почему мы?", "Программы", "Наша команда", "Отзывы"
3.	Система должна обеспечивать адаптивное отображение контента для устройств с шириной экрана от 320px до 1920px
4.	Система должна отображать контактные данные (номер телефона, ссылки на мессенджеры) в шапке и футере сайта

### Модуль "Каталог туров"
1.	Система должна отображать каталог туров в виде карточек, каждая из которых содержит: название, краткое описание, стоимость и кнопку "Узнать подробнее"
2.	Система должна предоставлять страницу детального просмотра тура с фотогалереей, полным описанием программы, условиями и формой подачи заявки

### Модуль "Взаимодействие с клиентом"
1.	Система должна предоставлять форму для подачи заявки с полями: "Имя" (текстовое поле, обязательное) и "Телефон" (поле с валидацией номера, обязательное)
2.	Система должна предоставлять ссылки для перехода в мессенджеры (Telegram, WhatsApp, VK) 

### Модуль "Авторизация и безопасность"
1.	Система должна предоставлять форму авторизации для сотрудников с полями "Логин" и "Пароль"
2.	Система должна проверять корректность учетных данных и предоставлять доступ к панели управления только после успешной аутентификации

### Модуль "Управление турами"
1.	Система должна предоставлять интерфейс для просмотра списка всех туров
2.	Система должна позволять добавлять новые туры, редактировать существующие (название, описание, стоимость, фотографии) и удалять туры
3.	Система должна предоставлять возможность изменять статус видимости тура (доступен/скрыт)
4.	Система должна валидировать вводимые данные (неотрицательная стоимость, заполненные обязательные поля)

### Модуль "Управление заявками"
1.	Система должна отображать список заявок от клиентов с информацией об имени, телефоне и дате подачи
2.	Система должна предоставлять возможность удаления обработанных заявок

### Модуль "Управление контентом"
1.	Система должна предоставлять интерфейс для редактирования информации о сотрудниках (ФИО, должность, описание, фотография)
2.	Система должна предоставлять интерфейс для управления отзывами (добавление, редактирование, удаление)


## Технические требования
- Два независимых проекта: клиентское приложение + админ-панель
- ES6 modules (import/export)
- TypeScript
- Запрещены: require, module.exports, var


## Use Case: Просмотр главной страницы
### Участники
Пользователь, Система отображения контента
### Предусловия
- Пользователь зашел на главную страницу
- Система загрузила основной контент
### Основной сценарий
1. Пользователь переходит на главную страницу
2. Система показывает основной баннер с предложениями
3. Пользователь листает страницу вниз
4. Система отображает блок "Наши преимущества"
5. Пользователь знакомится с преимуществами
6. Система показывает блок "Отзывы"
7. Пользователь просматривает отзывы
8. Система отображает форму подачи заявки
9. Пользователь заполняет и отправляет заявку (опционально)
10. Система показывает контакты для связи
11. Пользователь связывается по телефону или через соцсети (опционально)
### Результат
Пользователь ознакомился с услугами турагентства и может оставить заявку
### Альтернативный сценарий
- При ошибке загрузки система показывает страницу 404
- Пользователь обновляет страницу для повторной попытки


## Use Case: Подача заявки на тур
### Участники
Пользователь, Система отображения контента
### Предусловия
- Пользователь находится на сайте турагентства
- Пользователь ознакомился с описанием тура
### Основной сценарий
1.	Пользователь нажимает кнопку "Хочу в тур", или листает главную страницу ниже до блока подачи заявки, или нажимает кнопку "Оставить заявку"
2.	Система открывает модальное окно с формой заявки
3.	Пользователь заполняет поле "Имя"
4.	Пользователь заполняет поле "Телефон"
5.	Пользователь нажимает кнопку "Отправить заявку"
6.	Система проверяет корректность заполнения полей
7.	Система сохраняет заявку в базе данных
8.	Система показывает сообщение об успешно отправленной заявке
### Результат
Пользователь успешно отправляет заявку, система сохраняет данные и показывает подтверждение
### Альтернативный сценарий
1. Система обнаруживает, что поле "Телефон" заполнено некорректно
2. Система подсвечивает поле красным и показывает сообщение "Введите корректный номер телефона"
3. Пользователь исправляет номер телефона
4. Возврат к шагу 5


## Use Case: Просмотр каталога туров
### Участники
Пользователь, Система отображения контента
### Предусловия
- Пользователь находится на сайте
- Система загрузила актуальный каталог туров
### Основной сценарий
1.	Пользователь нажимает на пункт меню "Программы"
2.	Система отображает страницу с каталогом туров в виде карточек
3.	Система показывает название, краткое описание, стоимость и кнопку «Узнать подробнее» для каждого тура
4.	Пользователь просматривает доступные туры
### Результат
Пользователь ознакомился с доступными туристическими программами
### Альтернативный сценарий
1. Доступных туров нет
2. Система показывает сообщение "В настоящий момент туров нет"
